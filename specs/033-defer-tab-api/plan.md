# Implementation Plan: Defer Tab API Calls Until Tab Is Ready

**Branch**: `033-defer-tab-api` | **Date**: 2026-05-05 | **Spec**: spec.md

## Summary

The three dashboard tabs (Workforce, Industry, Recommendations) each dispatch their own API thunk. Currently all three fire immediately when the user lands on the dashboard. This feature defers each call until `useDashboardStatusPolling` confirms the back-end has finished processing that tab (`isWorkforceTabReady`, `isRecommendationTabReady`, `isIndustryTabReady`).

**Approach**:
- Gate `fetchWorkforce` and `fetchRecommendations` in the existing dispatch `useEffect` in `DashboardPage.tsx` using readiness flags plus a `!shouldPollDashboardStatus` bypass for non-polling scenarios.
- Add `enabled` parameter to `useIndustry` hook; `BenchmarkFinchPage` threads its existing `isReady` prop into it.
- Add/update tests for all three gating scenarios.
