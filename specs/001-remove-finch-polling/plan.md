# Implementation Plan: Consolidate Finch Connection Status

**Branch**: `001-remove-finch-polling` | **Date**: 2026-04-28 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-remove-finch-polling/spec.md`

## Summary

Remove `useFinchStatus` polling usage from UI flows and derive Finch connection state from assessment status (`assessmentData.assessmentType === "finch"`). This reduces redundant status calls and aligns all Finch gating logic to the existing assessment API response. Implementation updates hook consumers and test mocks, then deletes the obsolete status hook file.

## Technical Context

**Language/Version**: TypeScript 5.x + React 19  
**Primary Dependencies**: React Router v7, Redux Toolkit, existing assessment API client (`getAssessment`)  
**Storage**: N/A

**Constraints**: No behavior regressions in dashboard/additional-questions/recommendations; no new dependencies; preserve strict typing and existing routes  
**Scale/Scope**: 4 source files refactor + 1 source file removal + Finch-related test mock updates across page test suites

## Phase 2: Implementation Planning

5. Update and stabilize tests currently mocking `@/hooks/useFinchStatus`.
