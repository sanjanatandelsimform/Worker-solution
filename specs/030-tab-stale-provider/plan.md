# Implementation Plan: Tab Stale State & Provider-Aware Preparing Messages

**Branch**: `030-tab-stale-provider` | **Date**: 2026-05-01 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/030-tab-stale-provider/spec.md`

## Summary

Gate the "Preparing your dashboard" screen so it only appears for Finch-connected users (Change 1), and make the screen's description message reflect the user's payroll provider type — "24-36 hours" for automated providers vs "up to 2 weeks" for non-automated — with the messages stored in a shared constants file and the provider type exposed as `isAutomatedProvider` from the polling hook (Change 2).

## Technical Context

**Language/Version**: TypeScript 5 / React 19  
**Primary Dependencies**: React 19, Vite, `useDashboardStatusPolling` hook, `useAssessmentStatus` hook, React Testing Library + Vitest  
**Storage**: N/A (no new persistence; all data is derived from existing Redux state and API response)  
**Testing**: Vitest + React Testing Library (existing setup)  
**Target Platform**: Web / SPA (all modern browsers)  
**Project Type**: Single web application (`src/` root)  
**Performance Goals**: No change — all new logic is pure `useMemo` derivations with O(1) cost  
**Constraints**: No `any` types; strict TypeScript; all changes must be additive (backward-compatible props with defaults)  
**Scale/Scope**: 8 source files (1 new), 3-4 test files
