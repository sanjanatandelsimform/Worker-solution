# Implementation Plan: Finch Reauth Status Flag

**Branch**: `031-finch-reauth-status` | **Date**: 2026-05-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/031-finch-reauth-status/spec.md`

## Summary

The `dashboards/status` API is adding a new `finchConnectionStatus` field with values `"connected"`, `"reauth_required"`, `"disconnected"`, and `"pending"`. This feature consumes that field to:

1. Expose a boolean `isReauthRequired` flag from `useDashboardStatusPolling` (true only when value is `"reauth_required"`).
2. Conditionally render the existing "Reconnect to Finch" `DashboardCard` only when `isReauthRequired` is true (currently always rendered).
3. Add a `reconnectWithFinch` function to `useFinchConnect` that runs the Finch Connect flow but skips the post-success navigation to `/additional-questions`, so returning users stay on the dashboard.
4. Update tests for both hooks to cover the new behavior.
