# Implementation Plan: Finch Connect Loading Spinner

**Branch**: `021-finch-connect-loading` | **Date**: 2026-04-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/021-finch-connect-loading/spec.md`

## Summary

Add a full-screen loading spinner to `DashboardPage.tsx` that is displayed while the Finch connection flow is in progress (button click → session fetch → provider connection → code exchange). This is a single early-return guard using the existing `isFinchLoading` boolean and the existing `<LoadingSpinner>` component — no new files, no new dependencies, no hook changes.

## Technical Context

**Language/Version**: TypeScript 5.x + React 19  
**Primary Dependencies**: React 19, Tailwind CSS v4, existing `LoadingSpinner` component  
**Storage**: N/A  
**Testing**: Vitest + React Testing Library  
**Target Platform**: Web SPA (Vite)  
**Project Type**: Web  
**Performance Goals**: Spinner renders within one React paint cycle of button click (<16ms)  
**Constraints**: No new npm dependencies; no changes to `useFinchConnect` hook; no changes to tests  
**Scale/Scope**: 1 file, 1 early-return guard insertion (~4 lines of JSX)

### Phase 2: Implementation Breakdown

#### Step 1 — Add Finch loading guard in `DashboardPage.tsx`

In `src/pages/dashboard/DashboardPage.tsx`, immediately after the existing `isLoadingAssessment` guard, insert:

```tsx
if (isFinchLoading) {
  return <LoadingSpinner height={80} width={80} bgClass="bg-secondary" ariaLabel="oval-loading" />;
}
```

**No imports needed.** `LoadingSpinner` already imported; `isFinchLoading` already destructured.
